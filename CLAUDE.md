# duck-dash — Projektregeln

Minigame fuer eine Coding-Challenge. Diese Regeln gelten fuer jede Session in diesem Projekt.

## Git / Push (strikt)
- **Push-Ziel:** ausschliesslich `origin` = mein eigenes Repo `https://github.com/yunghundse/duck-dash.git`. Das Ziel niemals aendern, niemals woanders hin pushen.
- **Nach jedem abgeschlossenen Arbeitsschritt sofort committen und pushen:** `git add -A`, kurze klare Conventional-Commit-Message, `git push`. An den Fortschritt gekoppelt (kein Timer) — immer wenn wieder ein Stueck fertig ist.
- So landet der komplette Fortschritt fortlaufend und nachvollziehbar im eigenen Repo.

## Session-Start
- Zu Beginn jeder neuen Session zuerst `git log` und `git status` anschauen, kurz orientieren, dann nahtlos weiterbauen — weiterhin mit Push nach jedem Schritt.

## Stil
- Conventional Commits (feat:, fix:, docs:, chore:, refactor:). Commit-Messages auf Englisch.
- Antworten auf Deutsch, direkt und knapp.
